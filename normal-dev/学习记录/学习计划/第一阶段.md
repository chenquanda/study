### 专题3：Python的"坑"（1小时）

**输出位置**：`topic-03-pitfalls/`

新建 `pitfalls.py`，对以下每个"坑"，**先写出你猜的输出**（注释里写），再运行，对比结果：

```python
# 坑1：可变默认参数
def f(x, lst=[]):
    lst.append(x)
    return lst
# 你猜 f(1), f(2), f(3) 各返回什么？

# 坑2：闭包延迟绑定
funcs = [lambda: i for i in range(3)]
# 你猜 [f() for f in funcs] 输出什么？

# 坑3：is 和 ==
a = 256; b = 256
c = 257; d = 257
# 你猜 a is b 和 c is d 各是什么？为什么？

# 坑4：浅拷贝陷阱
import copy
x = [[1,2],[3,4]]
y = copy.copy(x)
y[0].append(99)
# 你猜 x 变成什么？

# 坑5：循环引用与GC
class A:
    def __init__(self):
        self.ref = None
a = A(); b = A()
a.ref = b; b.ref = a
del a; del b
# Python会回收吗？为什么？
```

每一个"坑"后面写一段**说明**：原理是什么、生产代码怎么避免。

### 方法论1：Python学习路径调整——"项目代码精读"

#### 背景

### 方法论2：关于"频繁查文档"

#### 一句话结论

**频繁查文档完全可以，而且必须可以。但要区分"查什么"。**

#### 三个查询层级

```
Level 1：查"它存不存在"
  例：Java的Map有没有方法直接获取所有值？→ 查到 values()
  → 【健康】，终身都要查

Level 2：查"它怎么用"
  例：HashMap.computeIfAbsent的参数是什么？
  → 【健康】，终身都要查

Level 3：查"它是什么概念"
  例：Map是干嘛的？什么时候该用？
  → 【不健康】，说明概念没建立，需要专项学习
```

**Level 1+2 永远要查。Level 3 才是要消灭的盲区。**

#### 关于"看能懂，敲不出来"

这是**完全正常**的现象，叫"识别 vs 回忆"差距：

- **识别**（看到代码能懂）：被动技能，容易获得
- **回忆**（凭空写出来）：主动技能，**只能通过手敲获得**

所以你的Java手写题计划是对的。但**不需要闭卷**，**带着小抄手敲**就行。

#### 三个具体策略

##### 策略1：建立"高频操作小抄"

- 不存在则计算：map.computeIfAbsent(key, k -> new ArrayList<>())
- 遍历：for (Map.Entry<K,V> e : map.entrySet())
- 转List：new ArrayList<>(map.values())

- ...

4. 写不出来的 → 加入小抄重点项 + 盲区清单

这个动作把"查"转化为"记"，效果远超单纯重复刷题。

### 方法论3：用AI做Code Review的标准流程

#### 一句话结论

**做完题后让AI给优化点，是工程师正确使用AI的姿势。强烈推荐，但必须按SOP执行。**

#### 适用场景

- ✅ 完成一道Java手写题后
- ✅ 完成一次Python项目代码精读后（让AI审查你的注释和理解）
- ✅ 完成一段Go练习代码后
- ❌ **不适用**：基线测试、Week 4的基线复测（这些必须保持纯净）

#### 标准操作流程（SOP）

##### Step 1：完成你的版本（关AI）

正常写到自己满意为止。**不要因为后面有AI审查就草率写**。

##### Step 2：自审（关AI，5分钟）

先自己审一遍——这一步很重要，否则审查能力会被外包给AI。

写 `self-review.md`：

```markdown
# 自审报告

## 我觉得写得不错的地方
1.
2.

## 我觉得可以更好的地方
1.
2.

## 我不确定的地方
1.
```

##### Step 3：AI Review（开AI）

**用这个标准prompt**（直接复制）：

```
我在练习[Java/Python/Go]编程能力，刚完成一道题：[简短描述]。

请你扮演严格的Code Reviewer，按以下规则给反馈：

1. 不要给我修改后的代码
2. 不要重写任何部分
3. 用清单形式给出最多5个优化点
4. 每个优化点说明：
   - 当前代码是什么样
   - 问题是什么（性能/可读性/规范/安全/...）
   - 优化方向（只说方向，不给代码）
   - 如果涉及API/语法，告诉我关键词，让我自己查文档
5. 严重程度分级：[严重] [建议] [可选]

我的代码：
[贴代码]

我的自审：
[贴self-review.md内容]
```

##### Step 4：处理AI反馈

对每个优化点：

1. **先停一下**，问自己："如果没看AI的话，我能想到吗？"
   - 想得到 → 标记【粗心】，下次注意
   - 想不到 → 标记【盲区】，加入盲区清单
2. **挑1-3条最有价值的**自己改（**不全改**）
3. **改的过程关AI**，自己查文档实现

##### Step 5：记录学习

新建 `[题目]-ai-review.md`：

```markdown
# AI Review笔记

## AI给了几条建议
1.
2.

## 我能自己想到的
1.

## 我想不到的（→ 盲区）
1.
2.

## 我采纳并实现了哪几条
1. xxx → 改进了xxx → 学到了xxx

## 我没采纳的（带原因）
1. xxx → 因为现阶段不需要 / 我不认同
```

#### ⚠️ 三条危险信号

出现以下情况立刻停止使用AI Review：

1. **跳过自审直接给AI** → 你在外包思考能力
2. **AI说什么改什么** → 你失去了主见
3. **改完不写笔记** → 这次学习等于没发生

#### 进阶用法：把AI变成"提问者"

适用于Week 3的Python专题深挖：

```
我刚写完一段代码（贴上）。请扮演面试官，提出5个关于这段代码的问题，难度递进：
- 1-2题：基础（变量、语法层面）
- 3-4题：进阶（设计、性能层面）
- 5题：原理（底层机制层面）

不要给答案。我会一题一题回答你，你判断对错并给反馈方向（不直接给答案）。
```

这个用法把AI变成你的私人教练——**逼你输出**，而不是被动接收。

# 🎓 第一个月学习指南（必读）

## 时间分配建议

按你的精力情况，**不强制每天学习**。建议：

- **工作日**：30-60分钟
  - 主要做副线（Python反向拆解 + Go阅读）
  - 微学习（盲区翻翻、文档读读）
- **周末**：2-4小时
  - 主线任务（Java手写题）
  - 专题深挖（Week 3的Python三专题）

**单次学习不要超过2小时**。学习是马拉松，不是冲刺。

## 三条底线规则

1. **关AI规则**：所有Java手写题、Python专题练习、基线测试都**严禁开AI**。允许查官方文档。
2. **盲区即时记录**：发现了不立刻记，5分钟后就忘。**用便签也行**，事后誊抄到清单。
3. **日志不断线**：哪怕一天只学5分钟，也要在 `03-daily-log.md` 写一行。**断线一次，意志消退50%**。

## 当你完成不了任务时

完成不了 ≠ 失败。**完成不了 = 信号**。问自己：

- 是任务太难？→ 拆小，今天只做一半
- 是精力不够？→ 改成微学习模式
- 是任务无聊？→ 换一道你更想做的题
- 是真的不想学？→ **写日志记录这个状态**，然后允许自己休息

**不要"明天补回来"**——补不回来。今天的事今天做掉一点，是一点。

## 当你想偷懒开AI时

允许自己**偷懒前先做一件事**：
- 在 `03-daily-log.md` 里写一行："XX时间，我想开AI，因为______"
- 写完之后，再决定开不开

90%的情况下，写完这句话你就不想开了。这是**元认知训练**，比任何技术学习都重要。

## 老师的最后叮嘱

第一个月不会让你"脱胎换骨"，但会做到三件事：

1. **把"关AI写代码"这个动作变成肌肉记忆**
2. **建立你自己的盲区数据库**（这是你后面所有学习的弹药库）
3. **让你看清自己的真实水平**（基线 → 复测的对比是关键）

**完成第一个月**，你才有资格谈"AI依赖怎么破"。在那之前，所有讨论都是空话。
